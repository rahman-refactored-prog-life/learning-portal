 // Shadows outer x\n    System.out.println(x);  // 20\n}\nSystem.out.println(x);  // 10\n```\n\n**Warning Signs**: Unexpected values, confusion about which variable is being used.\n\n**How to Avoid**: Use different names for variables in different scopes.\n\n---\n\n#### Pitfall 2: Autoboxing Performance Cost\n\n**The Trap**: Automatic conversion between primitives and wrappers is convenient but slow.\n\n**Example**:\n```java\n// Slow - creates 1 million Integer objects!\nInteger sum = 0;\nfor (int i = 0; i < 1_000_000; i++) {\n    sum += i;  // Autoboxing on every iteration\n}\n\n// Fast - uses primitive\nint sum = 0;\nfor (int i = 0; i < 1_000_000; i++) {\n    sum += i;\n}\n```\n\n---\n\n#### Pitfall 3: String Concatenation in Loops\n\n**The Trap**: String is immutable - concatenation creates new objects.\n\n**Example**:\n```java\n// Slow - creates 1000 String objects!\nString result = \"\";\nfor (int i = 0; i < 1000; i++) {\n    result += i;  // New String each time\n}\n\n// Fast - uses mutable StringBuilder\nStringBuilder sb = new StringBuilder();\nfor (int i = 0; i < 1000; i++) {\n    sb.append(i);\n}\nString result = sb.toString();\n```\n\n---\n\n### Anti-Patterns (Bad Practices)\n\n**Anti-Pattern 1: Magic Numbers**\n```java\n// ‚ùå BAD\nif (status == 1) { ... }\n\n// ‚úÖ GOOD\nfinal int STATUS_ACTIVE = 1;\nif (status == STATUS_ACTIVE) { ... }\n```\n\n**Anti-Pattern 2: Poor Variable Names**\n```java\n// ‚ùå BAD\nint x = 25;\ndouble y = 75000.50;\n\n// ‚úÖ GOOD\nint customerAge = 25;\ndouble annualSalary = 75000.50;\n```\n\n**Anti-Pattern 3: Unnecessary Type Conversions**\n```java\n// ‚ùå BAD\nint x = (int) 5; i < 1_000_000; i++) {\n    sum += i;  // Direct primitive operation, no objects created\n}\n```\n\n**When You Might See This**:\n- Code written by developers new to Java\n- When using collections and forgetting to convert back to primitives\n- Legacy code that hasn't been optimized\n\n**How to Refactor**:\n1. Use profiler to find other hotspots\n\n---\n\n#### Anti-Pattern 2: Using `float` for Currency\n\n**What it is**:\nStoring monetary values in `float` or `double`, leading to precision errors in financial calculations.\n\n**Example**:\n```java\n// ‚ùå ANTI-PATTERN\ndouble price = 0.1;\ndouble total = price * 3;\nSystem.out.println(total);  // Prints: 0.30000000000000004 (NOT 0.3!)\n```\n\n**Why it's bad**:\n- **Correctness**: Rounding errors accumulate, causing incorrect totals\n- **Legal**: Financial regulations require exact decimal arithmetic\n- **Trust**: Customers lose trust when charged incorrect amounts\n- **Bugs**: Subtle errors that only appear with certain values\n\n**Real-World Consequences**:\nA retail system using `double` for prices had rounding errors that cost the company $50,000 in incorrect charges before being discovered.\n\n**Better Approach**:\n```java\n// ‚úÖ BETTER PATTERN\nimport java.math.BigDecimal;\n\nBigDecimal price = new BigDecimal(\"0.10\");\nBigDecimal total = price.multiply(new BigDecimal(\"3\"));\nSystem.out.println(total);  // Prints: 0.30 (EXACT!)\n\n// Or store cents as integers\nint priceInCents = 10;  // $0.10\nint totalInCents = priceInCents * 3;  // 30 cents = $0.30\n```\n\n**When You Might See This**:\n- Quick prototypes that made it to production\n- Code by developers unfamiliar with floating-point issues\n- Systems migrated from languages with decimal types\n\n**How to Refactor**:\n1. Add tests with MAX_VALUE and MIN_VALUE\n\n---\n\n### Debugging Strategies\n\n#### Strategy 1: Print Variable Types and Values\n\n**When to use**: When you're confused about type conversions or unexpected values\n\n**How to do it**: \n1.
